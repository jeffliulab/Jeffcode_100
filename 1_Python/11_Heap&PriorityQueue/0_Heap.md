1、基本概念
Heap（Priority Queue）是

2、Max Heap / Min Heap
Heap分为MaxHeap/MinHeap。
MaxHeap满足：根节点 ≥ 子节点
MinHeap满足：根节点 ≤ 子节点
Heap 不需要 根节点与整棵子树的所有节点比较，只看直接子节点，这点和BST完全不同。换句话说，堆是局部有序的，而 BST 是全局有序的。

3、传统堆（Binary Heap）一定是二叉树，并且是一棵 完全二叉树。
完全二叉树的特点是：从上到下、从左到右逐层填满节点，最后一层的节点靠左对齐。
堆（Binary Heap）是完全二叉树的一种，节点按从上到下、从左到右的顺序依次填充，所以必须将上一层填满后，才能继续存放下一层的节点。
但并不是所有堆都必须是二叉树。扩展的堆（如 d-ary Heap）可以有 d 个子节点。

4、传递关系：
如果堆从索引 1 开始存储元素，而不是 0，那么父节点和子节点的关系计算会更简单：
    
    父节点索引 = 子节点索引 i // 2
    
    左子节点索引 = 父节点索引 i * 2
    右子节点索引 = 父节点索引 i * 2 + 1

因此，heap通过array的形式存储会非常高效。


4、sift up 向上调整：
插入到堆中的新元素确实是直接添加到 堆的末尾，然后通过 与其父节点的比较 来调整位置，这个过程叫做 向上调整（sift up），从而恢复堆的性质。

5、sift down 向下调整：
向下调整的过程与向上调整相反，用于维护堆的性质，特别是在删除堆顶元素或其他节点后需要重新调整堆时。
向下调整的目标是将一个不符合堆性质的节点“下沉”到正确的位置，确保堆的性质得以恢复。
